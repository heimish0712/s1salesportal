STEP18 - 계약조건 숫자형 저장/선택 범위 확장 보고서

기준선
- 영업전산포탈_STEP17_스타일노출_사이드바겹침수정.zip

수정 목적
- 영업관리대장 마스터시트 U/W/X 열의 데이터 형식 호환 유지
  - U: 계약단위 = 숫자만 저장(1~12)
  - W: 유지점검 = 숫자만 저장
  - X: 성능점검 = 숫자만 저장
- 포털 화면에서는 계약단위를 1~12개월까지 선택 가능하게 확장
- 신규 고객 기본 계약단위는 12개월 유지

수정 파일
- 01_Schema.gs
- 05_CustomerService.gs
- Component_CustomerDetail.html
- Component_CustomerSearch.html

주요 반영
1. 계약단위 옵션 확장
- 기존: 6개월, 12개월
- 변경: 1개월~12개월 전체 선택 가능
- select option value는 숫자 문자열(예: 12), label은 화면 표시용(예: 12개월)

2. 유지점검/성능점검 숫자형 저장
- 화면 표시: 0회~12회
- 저장 value: 0~12 숫자 문자열
- 서버 write 시에는 Number 타입으로 setValue/setValues 처리

3. 서버 저장 정규화
- saveCustomerDetail
- saveCustomerDetailFast
- saveRegistrationCustomer
위 경로 모두 저장 전 contractUnit/maintenance/performance를 숫자형으로 정규화.

4. 견적 계산 정합성
- calculateQuoteDiscount에서 계약단위 숫자값을 월 수로 직접 계산.
- 1~12개월 모두 선임비 계산에 반영.
- 계산 결과 contractUnit도 숫자형 value로 반환.

5. 신규등록 기본값
- 계약단위: 12
- 관리자선임: 선임
- 유지점검: 2
- 성능점검: 1
- 부가세: 별도

6. 기존 데이터 호환
- 기존 값이 '12개월', '6개월', '1년', '반년', '2회'처럼 들어와도 화면 선택값과 저장값은 숫자로 정규화.
- 펼침 카드 표시에서는 숫자값을 12개월/2회처럼 읽기 쉬운 형식으로 표시.

검증
- 모든 .gs 문법 검사 통과
- 모든 .html script 문법 검사 통과
- top-level function 중복 0건
- 서버/자료발송/영업지원/오늘할일/드라이브반영 기존 로직 변경 없음
