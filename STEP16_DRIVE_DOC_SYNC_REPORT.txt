STEP16 - 계약종합관리 드라이브 반영 기능 추가

기준선
- STEP15_발주여부조회속도캐싱개선.zip 기준

요청 반영
1. 계약종합관리 새로고침 버튼 옆에 [드라이브 반영] 버튼 추가
   - id: contractDriveSyncBtn
   - data-admin-only="Y"
   - 눈에 띄는 주황 계열 버튼 스타일 적용

2. 권한 제한
   - 클라이언트: canUseAdminHome 권한이 없으면 버튼 숨김/차단
   - 서버: syncContractCompleteDocsFromDriveP270()에서 getPortalCurrentPermission_().canUseAdminHome 검증
   - 서무/admin 계정만 실행 가능

3. Drive 스캔 로직
   - 공유 드라이브 폴더명: S1 고객사 파일 관리
   - optional ID 상수: PORTAL_CUSTOMER_FILE_ROOT_FOLDER_ID_P270
   - 폴더명 앞 숫자 = 고객번호
   - 파일명 앞 숫자 = 계약번호
   - 고객번호와 계약번호가 모두 수주확정/계약완료 행과 맞는 경우만 반영

4. 반영 컬럼
   - 사업자등록증 파일명 감지 -> 사업자등록증 저장 = 저장
   - 선임신고서/위임장 파일명 감지 -> 선임신고서 저장 = 저장
   - 계약서/용역신청서/표준계약서 파일명 감지 -> 계약서 저장 = 저장

5. 웹앱 즉시 반영
   - 서버 반영 성공 후 updatedRows를 반환
   - 클라이언트 contractCompleteRowsV69에 즉시 patch 후 재렌더
   - 계약종합관리 캐시 clearContractCompleteCacheV69_() 처리

6. 기존 호환성
   - 내부 key orderMailSaved는 기존 코드 호환을 위해 유지
   - 헤더 후보에 선임신고서 저장/발주메일 저장을 모두 포함
   - UI 라벨은 선임신고서 저장으로 표시

수정 파일
- 16_ContractCompleteService.gs
- Component_ContractComplete.html
- Index.html
- Styles.html

주의
- DriveApp.getFoldersByName('S1 고객사 파일 관리')로 폴더를 찾습니다.
- 공유 드라이브 환경에서 이름 검색이 실패하면 PORTAL_CUSTOMER_FILE_ROOT_FOLDER_ID_P270에 폴더 ID를 넣으면 됩니다.
- 서류가 없는 경우 기존 저장값을 지우지 않습니다. 발견된 서류만 '저장'으로 반영합니다.
