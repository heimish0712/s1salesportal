1) 영업지원 요청 작성 권한/요청자 선택 규칙
- 영업지원 요청 작성 권한은 권한_DB에 등록된 활성 사용자 전체로 확대했습니다.
- 단, 완료 처리/전체 열람/관리자 홈 권한은 기존 정책을 유지했습니다.
- 서무/책임/관리자성 계정(canUseAdminHome 또는 canCompleteSupport)은 요청자를 아래 목록에서 선택할 수 있습니다.
  문형진 / 방수원 / 박새봄 / 김경아 / 최보람 / 이옥희
- 일반 사용자는 요청자 옵션이 현재 사용자 본인 1명만 표시됩니다.
- 프론트 select 제한만으로 끝내지 않고 savePortalSupportRequest 저장 시 assertPortalSupportRequesterAllowedP250_()로 서버 검증도 추가했습니다.
- 기존 support detail ScriptCache가 다른 사용자 requesterOptions를 반환하지 않도록 캐시 hit 시 permission/requesterOptions/currentUser를 현재 사용자 기준으로 재계산합니다.
- 권한 캐시 prefix를 P250으로 변경해 기존 5분 캐시가 canWriteSupport 변경을 붙잡지 않게 했습니다.

2) 고객 상세검색 -> 자료발송 이동 시 새 작업창 추가
- openSendForCustomer(rowNo)를 변경했습니다.
- 고객 상세검색의 자료발송 버튼으로 넘어올 때 기존 활성 발송 작업창을 덮어쓰지 않고 새 발송 작업창을 생성합니다.
- 이미 5개가 열려 있으면 createdAt/order 기준 가장 오래된 작업창을 초기화해 새 고객으로 교체합니다.
- 발송 화면 내부에서 고객을 검색해 선택할 때의 기존 중복 고객 방지 로직은 유지했습니다.
- 외부 이동(openSendForCustomer)에서만 selectSendCustomer(rowNo, { skipDuplicateCheck: true })를 사용합니다.

3) 자료발송 하단 버튼 데크 축소
- .send-fixed-footer를 전체 폭 sticky bar가 아니라 버튼 묶음 크기에 맞는 width:max-content 캡슐형 영역으로 변경했습니다.
- 기존 버튼 3개와 기능은 유지했습니다.
- .send-workspace padding-bottom도 과도하지 않게 줄였습니다.

4) 자료발송 모든 체크 해제 버튼 추가
- 발송자료 빠른 선택 영역에 [모든 체크 해제] 버튼을 추가했습니다.
- clearAllSendFileChecksP250_()에서 체크박스를 모두 해제하고 selectedFileKeys/reviewSession/preview/workspace 상태를 같이 갱신합니다.
