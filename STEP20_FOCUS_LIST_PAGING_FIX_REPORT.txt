STEP20 - 중점고객관리 리스트 표시/20개 페이징 수정 보고서

기준선:
- STEP19_중점고객관리메뉴추가

문제:
- 중점고객관리 화면에서 전체 통계는 계산되지만, 검색조건이 없는 기본 상태에서도 목록이 0건으로 표시됨.
- 리스트 표시 단위가 50개였음.

원인:
- 서버 필터 정규화 과정에서 담당자/진행현황 기본값 ALL이 normalizeFocusTextP290_()를 거치며 'all'로 변환됨.
- 이후 필터 조건에서는 'ALL'과 비교하고 있어, 'all'이 실제 필터값으로 간주됨.
- 결과적으로 row.salesRep/status가 'all'인 고객만 찾게 되어 전체 목록이 비는 현상이 발생함.

수정:
1. 23_FocusCustomerService.gs
   - PORTAL_FOCUS_DEFAULT_PAGE_SIZE_P290: 50 -> 20
   - normalizeFocusFilterSentinelP291_() 추가
   - salesRep/status 필터의 ALL sentinel을 대소문자 보존 처리
   - pageSize 기본값 20개 적용

2. Component_FocusCustomer.html
   - 클라이언트 기본 pageSize: 50 -> 20
   - pager 기본 표시 기준 20개로 수정
   - 조회 결과 통계가 0인 경우에도 정확히 0으로 표시되도록 보정
   - 빈 목록 메시지 개선

기대 동작:
- 검색조건이 없으면 전체 중점고객 목록이 기본으로 표시됨.
- 기본적으로 제외 고객은 숨김 상태이며, 제외 포함 체크 시 제외까지 표시됨.
- 한 페이지에 20개씩 표시되고 이전/다음 버튼으로 이동.

검증:
- 수정 파일 문법 검사 통과
- 수정 파일 내부 top-level function 중복 없음
- 기존 고객검색/자료발송/영업지원/오늘할일/계약종합관리 로직 변경 없음
