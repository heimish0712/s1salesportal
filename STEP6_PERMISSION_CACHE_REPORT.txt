영업전산포탈 STEP6 - 서버 권한 조회 캐시 추가

기준 파일: STEP5_검색캐시무효화정리
수정 파일: 15_PermissionService.gs

반영 내용
1. getPortalCurrentPermission_()에 ScriptCache 기반 짧은 권한 캐시 추가
   - active user email 기준 캐시 키 생성
   - 등록/활성 권한은 300초 캐시
   - 미등록/비활성 guest 결과는 60초만 캐시
   - active user email이 비어 있으면 권한_DB를 열지 않고 즉시 guest 차단

2. 권한 정책 변경 없음
   - 기존 buildPortalPermissionFromRow_(), canPortal..., assertPortal... 로직 유지
   - getPortalCurrentPermission_()의 시트 조회 앞단에 캐시만 추가

3. 캐시 무효화 기능 추가
   - clearPortalPermissionCache(): cacheBust 값을 바꿔 기존 권한 캐시 즉시 무효화
   - setupPortalPermissionSheet(), resetPortalPermissionSheetForPatchS() 실행 시 cacheBust 자동 갱신

4. 권한_DB 직접 수정 감지 트리거 추가
   - installPortalPermissionCacheBusterTriggerP230() 실행 시 웹앱 DB 스프레드시트 onEdit 트리거 설치
   - onPortalPermissionSheetEditP230(e): 권한_DB 수정 시 cacheBust 갱신
   - 트리거를 설치하지 않아도 TTL 5분 뒤에는 자연 갱신됨

5. 진단 함수 추가
   - getPortalPermissionCacheStatus(): 현재 계정 기준 캐시 키, hit 여부, cacheBust, TTL 확인

검증
- 모든 .gs 문법 검사 통과
- 모든 .html script 문법 검사 통과
- 15_PermissionService.gs top-level function 중복 0건

주의
- 권한_DB를 수동으로 수정했는데 캐시 무효화 트리거를 설치하지 않은 경우, 최대 5분 동안 기존 권한이 유지될 수 있음
- 즉시 반영이 필요하면 clearPortalPermissionCache()를 실행하거나 installPortalPermissionCacheBusterTriggerP230()를 1회 설치해야 함
