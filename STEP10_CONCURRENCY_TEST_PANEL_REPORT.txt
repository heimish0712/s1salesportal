안전장치
1. 실제 메일 발송 함수는 호출하지 않음.
2. 관리자/서무 권한(canUseAdminHome)이 없으면 화면과 서버 함수 모두 차단.
3. 영업지원/오늘할일 테스트 데이터는 [CONCURRENCY_TEST:실행ID] 마커 사용.
4. 영업지원 테스트는 고객번호를 비워 마스터 메모 반영을 피함.
5. cleanupPortalConcurrencyTestRunP240()로 실행ID 기준 테스트 데이터 정리 가능.
6. 고객 메모 테스트는 rowNo 입력 시에만 실행되며, 실행 전 원본 메모를 포털_동시성테스트상태 시트에 저장함.
7. 고객 메모 테스트 cleanup 시 저장된 원본 메모로 복구함.

테스트 시나리오
1. SUPPORT_CREATE
   - savePortalSupportRequest()를 여러 요청에서 동시에 호출
   - 접수번호 중복, rowNo 중복, Lock timeout 여부 확인
   - cleanup 시 요청업무에 marker가 있는 테스트 행만 삭제

2. TODAY_UPSERT
   - savePortalTodosForDate()를 여러 요청에서 동시에 호출
   - 각 요청은 고유 할일ID를 사용
   - baseTaskIds=[]를 명시해 기존 다른 할일 삭제를 방지
   - cleanup 시 marker가 있는 오늘할일만 삭제여부=Y 처리

3. CUSTOMER_MEMO
   - 입력한 rowNo에 대해 saveCustomerMemoFast()를 여러 요청에서 동시에 호출
   - 실제 고객 메모를 바꾸므로 TEST 고객 rowNo에서만 사용 권장
   - 실행 전 원본 메모를 상태 시트에 저장
   - cleanup 시 원본 메모로 복구

사용 방법
1. 새 배포 후 관리자/서무 계정으로 포털 접속
2. 좌측 하단 [동시성 테스트] 클릭
3. 동시 요청 수 5 또는 10부터 시작
4. 영업지원요청 테스트 실행
5. 오늘할일 테스트 실행
6. 필요 시 TEST 고객 rowNo를 입력하고 고객 메모 테스트 실행
7. 실행 후 [마지막 실행ID 정리]로 테스트 데이터 cleanup

권장 테스트 순서
- 1차: 동시 요청 수 5, 랜덤 지연 150ms
- 2차: 동시 요청 수 10, 랜덤 지연 150ms
- 3차: 동시 요청 수 20, 랜덤 지연 300ms
- 고객 메모 테스트는 TEST 고객 rowNo로만 별도 실행

회귀 방지
- 기존 메일 발송, 고객 저장, 영업지원, 오늘할일, onEdit 함수는 삭제/이름변경하지 않음.
- 기존 운영 함수 호출부는 수정하지 않음.
- 테스트 패널은 신규 route/section/server API로 분리.
