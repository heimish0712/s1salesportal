반영 내용
1. 고객 상세검색 펼침 영역의 고객사 기본 정보 표에서 `고객등급` 표시 위치를 `발주여부`로 교체.
2. 발주여부 표시 규칙:
   - 수주확정/계약완료 시트에 동일 고객번호가 있으면 `O`
   - 없으면 `X (발주하기)`
   - 상세정보가 서버에서 아직 보강되기 전에는 `확인중`
3. `발주하기` 클릭 시 커스텀 팝업 표시:
   - 문구: `(고객사명)의 계약이 확정되었습니까?`
   - 버튼: 예 / 취소 / X
   - 취소 또는 X는 팝업만 닫음
4. 예 클릭 시 `createPortalCustomerOrderFromSearchP250()` 서버 함수 호출.
5. 서버 처리:
   - Script Lock 안에서 실행
   - 고객번호/rowNo로 마스터시트 대상 고객 재확인
   - 사용자 권한 검증
   - 수주확정/계약완료 시트에서 기존 동일 고객번호 발주건 중복 확인
   - 기존 발주건이 있으면 새 행을 만들지 않고 기존 계약번호 반환
   - 신규 발주건이면 계약번호 = 현재 최대 계약번호 + 1
   - 데이터가 있는 마지막 행 아래에 신규 행 작성
   - 마스터시트 진행현황을 `발주완료`로 보정
   - 계약종합관리 캐시 제거
6. 완료 팝업:
   - `(고객사명)의 발주번호(계약번호)는 (계약번호)번 입니다.`
7. 수주확정/계약완료 시트명 호환:
   - Google Sheets 기준 `수주확정/계약완료`
   - XLSX 참고파일 기준 `수주확정계약완료`
   - 두 이름 모두 탐색하도록 보강

수주확정/계약완료 이관 매핑
- 계약번호: 최대 계약번호 + 1
- 고객번호: 마스터 고객번호
- 계약일자: 실행일
- 지역/도시: 고객사 주소 및 지역구분 기준 추정
- 계약담당자: 영업담당자
- 고객사명: 회사명
- 담당자 성함: 고객사 담당자
- 전화번호: 대표전화번호 / 직통번호 병합
- 이메일 주소: 담당자 이메일 주소
- 연면적: 연면적
- 관리등급: 관리등급
- 계약가: 최종 견적가
- VAT: 부가세
- 수행사: 수행사, KJ는 케이제이로 보정
- 사업자등록번호: 사업자등록번호
- 대표자명: 대표자명
- 업종: 업종/업태/종목 후보값
- 고객사 주소: 고객사 상세 주소
- 계약기간: 명시 계약기간이 있으면 반영, 없으면 공란
- 비상주선임: 선임이면 계약단위 개월 수, 기본 12
- 유지점검: 유지점검 횟수 숫자
- 성능점검: 성능점검 횟수 숫자
- 청구 등 메모/예정일/완료여부: 초기 발주 단계에서는 공란

주의
- 실제 수주확정/계약완료 시트의 문서 저장 여부(사업자등록증 저장, 발주메일 저장, 계약서 저장)는 실제 파일 저장 여부와 직결될 수 있어 자동으로 `저장`을 쓰지 않았습니다.
- 고객등급 편집 기능 자체는 상세정보수정 모달에 남아 있으며, 펼침 표 표시만 발주여부로 교체했습니다.
